Laboration 1

Såhär fungerar internettrafik i en webbläsare

Internettrafik är uppdelat i flera olika lager. I botten har man ett transportlager som kallas TCP. Med TCP upprättar man en anslutning till en annan server som sedan kan användas för att skicka data mellan din dator och servern via olika internetprotokoll som körs ovanpå TCP. Ett exempel på ett sådant protokoll är TLS som används för att kryptera datatrafiken i en TCP-uppkoppling. Ett annat sådant protkoll är DNS. DNS används för att översätta en textbaserad webbadress såsom "www.google.com" till ett en numerisk adress som datorn förstår. En sådan adress kallas för ett IP-nummer. I webbläsaren används ännu ett protokoll som körs ovanpå TCP - HTTP. I HTTP arbetarm man utefter dynamiken begäran eller request och svar eller response. Klienten (i det här fallet din webbläsare) skickar en request till en server som svarar med en response. En request är typiskt att man ber servern om en html-sida (t.ex genom att klicka på en länk). Server svarar då, förutsatt att allt gått som det ska med en response som innehåller den HTML och de övriga filer browsern behöver för att visa sidan.
